使用约束

约束为Grails提供了声明性DSL，用于定义验证规则，模式生成和CRUD生成元数据。例如，请考虑以下约束：

class User {
    ...

    static constraints = {
        login size: 5..15, blank: false, unique: true
        password size: 5..15, blank: false
        email email: true, blank: false
        age min: 18
    }
}

有关详细信息，请参阅链接上的用户指南主题： Constraints

全局约束

您可以在 grails-app/conf/application.groovy 中全局应用约束，如下所示：

grails.gorm.default.constraints = {
    '*'(nullable: true, size: 1..20)
}

通配符表示约束适用于所有属性。 您还可以定义共享约束：

grails.gorm.default.constraints = {
    myShared(nullable: true, size: 1..20)
}

可以在您的课程中重用：

class User {
    ...

    static constraints = {
        login(shared: "myShared")
    }
}

共享约束

全局约束是在不同类之间（例如，在域类和命令对象之间）共享约束的一种方式。 这不再是唯一的方法。Grails2为 constraints 块引入了新语法，使您可以直接从另一个类重用约束。

假设您有一个像这样的域类：

class User {
    String firstName
    String lastName
    String passwordHash

    static constraints = {
        firstName blank: false, nullable: false
        lastName blank: false, nullable: false
        passwordHash blank: false, nullable: false
    }
}

然后，您要创建一个命令 UserCommand ，该对象共享域类的某些属性和相应的约束。 您可以使用 importFrom() 方法来实现：

class UserCommand {
    String firstName
    String lastName
    String password
    String confirmPassword

    static constraints = {
        importFrom User

        password blank: false, nullable: false
        confirmPassword blank: false, nullable: false
    }
}

这将从 User 域类中导入所有约束，并将其应用于 UserCommand 。导入将忽略源类( User )中任何在导入类( UserCommand )中没有相应属性的约束。 在上述情况下，仅将’firstName’和’lastName’约束导入到 UserCommand 中。

如果您想更多地控制导入哪些约束，请使用 include 和 exclude 命名参数。 这两个都接受与源约束中的属性名称匹配的简单或正则表达式字符串列表。 因此，例如，如果您只想导入 'lastName' 约束，则可以使用：

...
static constraints = {
    importFrom User, include: ["lastName"]
    ...
}

或者如果您想要所有以’Name’结尾的约束：

...
static constraints = {
    importFrom User, include: [/.*Name/]
    ...
}

当然，exclude 的作用相反，指定应不导入哪些约束。

children(maxSize: 25)

login(unique: true)

将自定义验证添加到字段。

脚手架

一些约束条件对持久性没有影响，但是可以自定义支架。 在您的域中包含UI信息通常不是一个好习惯，但是如果您广泛使用Grails的脚手架，那将是非常方便的。

约束 说明

display

布尔值，用于确定属性是否显示在支架视图中。 如果为 true （默认值），则显示属性。

editable

布尔值，用于确定是否可以从支架视图中编辑属性。 如果为false，则关联的表单字段以只读模式显示。

format

指定一种接受格式的显示格式，例如日期。 例如，'yyyy-MM-dd'。

password

布尔值，指示这是否是属性，应与密码字段一起显示。 仅适用于通常与文本字段一起显示的字段。

widget

控制使用什么窗口小部件来显示属性。 例如，'textarea’将强制脚手架使用<textArea>标签。

程序访问

您可以通过访问域类的 constrainedProperties 静态属性，在另一个上下文中以编程方式访问域的约束。 该属性是 Map<String, ConstrainedProperty > 的实例。

class User {
    String firstName
    String middleName

    static constraints = {
        firstName blank: false, nullable: false
        middleName blank: true, nullable: true
    }
}

在上面的例子中，访问 User.constrainedProperties.firstName.blank 会产生 false ，而 User.constrainedProperties.middleName.blank 会产生 true 。

命令对象

访问命令对象上的约束略有不同。通过访问实现 Validateable类的 constraintsMap 静态属性，可以在另一个上下文中以编程方式访问命令对象的约束。该属性是 Map<String, ConstrainedProperty > 的一个实例.

class User implements Validateable {
    String firstName
    String middleName

    static constraints = {
        firstName blank: false, nullable: false
        middleName blank: true, nullable: true
    }
}

在上面的示例中，访问 User.constraintsMap.firstName.blank 将产生 false ，而 User.constraintsMap.middleName.blank 将产生 true 。
