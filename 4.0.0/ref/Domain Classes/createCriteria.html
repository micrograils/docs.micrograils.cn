createCriteria

目的

创建并返回Grails的 HibernateCriteriaBuilder实例，该实例可用于构造条件查询。

单元测试使用的是不带Hibernate的GORM内存实现，因此， createCriteria() 返回 CriteriaBuilder，而不是 HibernateCriteriaBuilder。

例

可以从HibernateCriteriaBuilder（如本文档的以下示例中）静态导入用于构建条件的条件方法（例如，和，或，之间等），或写为“ c.like（…​）”，“ c.between（…​）“等

def c = Account.createCriteria()
def results = c.list {
    like("holderFirstName", "Fred%")
    and {
        between("balance", 500, 1000)
        eq("branch", "London")
    }
    maxResults(10)
    order("holderLastName", "desc")
}

要使用分页，您将不得不进行另一个查询以检索匹配结果的总数。更好的方法是将分页值作为参数传递给criteria方法，如下所示：

def c = Account.createCriteria()
def results = c.list (max: 10, offset: 10) {
    like("holderFirstName", "Fred%")
    and {
        between("balance", 500, 1000)
        eq("branch", "London")
    }
    order("holderLastName", "desc")
}

由于该查询包含分页参数（最大和偏移），因此将返回 PagedResultList，其中包含 getTotalCount() 方法，返回用于分页的匹配记录总数。两个查询仍在运行，但它们为您运行，并且结果和总数在 PagedResultList 中合并。

println "Rendering ${results.size()} Accounts of ${results.totalCount}"

描述

标准查询是一种类型安全的高级查询方式，它使用Groovy构建器构造潜在的复杂查询。这是使用StringBuilder动态构建HQL查询的更好的选择。有关使用说明，请参阅 Criteria上的用户指南部分。

方法参考：

方法 描述

list

默认方法；返回所有匹配的行.

get

返回唯一的结果，即仅一行。标准必须以这种方式形成，即它仅查询一行。此方法不应与仅第一行的限制混淆。

scroll

返回可滚动的结果集

listDistinct

如果使用子查询或关联，则在结果集中可能多次出现同一行。在Hibernate中，可以执行“ CriteriaSpecification.DISTINCT_ROOT_ENTITY”。在Grails中，只需使用此方法就可以做到。
listDistinct（）方法不能与分页选项maxResult和firstResult一起使用。如果您需要分页获得不同的结果，我们建议您现在使用HQL。您可以从 this blog post中找到更多信息。 如果不使用方法名称调用构建器 
c { ... }

list() 方法将被自动调用。换句话说，它相当于

c.list { ... }

集合属性的大小不等于特定值的情况

sqlRestriction

使用任意SQL修改结果集

sqlRestriction "char_length(first_name) = 4"

使用动态查找器，您可以访问诸如max，sort等选项。这些条件也可用于条件查询，但是它们具有不同的名称：

Name 描述 例

order(String, String)

指定排序列（第一个参数）和排序顺序（“ asc”或“ desc”）。

order "age", "desc"

firstResult(int)

指定结果的偏移量。值为0将返回所有记录，直到指定的最大值。

firstResult 20

maxResults(int)

指定要返回的最大记录数。

maxResults 10

cache(boolean)

指示是否应缓存查询（如果启用了查询缓存）。

cache 'true'

条件也支持预测的概念。投影用于更改结果的性质。例如，以下查询使用投影来计算每个 Account 存在的不同 branch 名称的数量：

def c = Account.createCriteria()
def branchCount = c.get {
    projections {
        countDistinct "branch"
    }
}
